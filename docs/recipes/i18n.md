# Recipe: internationalization (i18n)

Helios ships English-only. This is the shortest path to multi-language
support using `react-i18next`, with the initial locale picked up from the OS
via `@tauri-apps/plugin-os` (already a dependency, used today for
`app_info`'s platform string).

## 1. Install

```bash
pnpm add i18next react-i18next i18next-browser-languagedetector
```

## 2. Add translation resources

Create `src/lib/i18n/` with one JSON file per locale, e.g.:

```
src/lib/i18n/locales/en.json
src/lib/i18n/locales/fr.json
```

## 3. Initialize i18next, seeded from the OS locale

```ts
// src/lib/i18n/index.ts
import { locale as osLocale } from '@tauri-apps/plugin-os'
import i18next from 'i18next'
import { initReactI18next } from 'react-i18next'
import en from './locales/en.json'
import fr from './locales/fr.json'

const detected = (await osLocale()) ?? 'en'

await i18next.use(initReactI18next).init({
  resources: { en: { translation: en }, fr: { translation: fr } },
  lng: detected.split('-')[0],
  fallbackLng: 'en',
  interpolation: { escapeValue: false },
})

export { i18next }
```

`osLocale()` is only meaningful inside the Tauri shell — guard with the
existing `isTauri()` helper (`src/lib/tauri.ts`) and fall back to `'en'` in
dev/test (jsdom has no OS locale).

## 4. Wire it up

Import `./lib/i18n` once, before the render call in `src/main.tsx` (top of the
file, alongside the existing font imports), and add it as a dependency of the
`os:default` permission already granted in
`src-tauri/capabilities/default.json` (no capability change needed — `locale`
is part of `os:default`).

## 5. Persist a manual override

If you want a user-facing language switcher (Settings page), store the choice
the same way `theme` is stored today: add a field to `Settings` in
`src-tauri/src/commands/settings.rs`, bump `CURRENT_SCHEMA_VERSION`, and call
`i18next.changeLanguage(lang)` when it changes — mirror `applyTheme` in
`src/app/providers.tsx`.

## 6. Tests

Vitest components using `useTranslation()` need i18next initialized before
render — initialize once in `src/test/setup.ts` with a synchronous
`en`-only config so tests don't depend on OS locale detection.
