# Helios Enterprise Tauri Starter — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform the fresh create-tauri-app scaffold into the enterprise-grade Tauri 2 starter defined in `docs/superpowers/specs/2026-07-06-enterprise-tauri-starter-design.md`.

**Architecture:** `package.json` is the single version source; release-please + tauri-action in one workflow produce signed-updater GitHub releases; tauri-specta generates the typed IPC boundary; the frontend is feature-foldered React Router v7 with a three-layer state model (TanStack Query / Zustand / tauri-plugin-store); a Claude Code agent pack (CLAUDE.md, rules, skills, settings) encodes every procedure.

**Tech Stack:** Tauri 2, React 19, Vite, TypeScript, pnpm, Tailwind v4 + shadcn/ui, React Router v7 (hash), TanStack Query, Zustand, tauri-specta, Biome v2, Vitest, lefthook + commitlint, release-please, tauri-action, cargo-deny, CodeQL, Renovate.

## Global Constraints

- Repo: `https://github.com/itsJeremyMax/helios`, branch `main`, squash-merge only. All commits Conventional Commits (they seed the changelog).
- Version lives ONLY in `package.json`; `src-tauri/tauri.conf.json` must contain `"version": "../package.json"`; release-please bumps `src-tauri/Cargo.toml` via extra-files. Never hand-bump.
- `src/bindings.ts` is GENERATED by tauri-specta — never hand-edit; it is committed and CI fails if stale.
- Frontend never imports `invoke` from `@tauri-apps/api` directly — all IPC through `src/bindings.ts` (re-exported via `src/lib/ipc/`).
- Rust commands: return `AppResult<T>`; never `panic!`/`unwrap` in command paths; every command carries `#[tauri::command]` + `#[specta::specta]` and is registered in the specta builder in `lib.rs`.
- Any UI implementation step MUST begin by invoking the `frontend-design:frontend-design` skill. Fonts: bundled `@fontsource-variable/space-grotesk` (headings), `@fontsource-variable/manrope` (body/UI), `@fontsource-variable/jetbrains-mono` (mono) — never system-default stacks, never CDN fetches.
- Features are self-contained under `src/features/<name>/` with a public `index.ts`; features never import each other; route registration in `src/app/routes.tsx` is the single integration point.
- Persistent client state goes through the tauri-store-backed Zustand adapter (Task 12) — never `localStorage`.
- pnpm only (no npm/yarn). Rust toolchain pinned by `rust-toolchain.toml` (stable). pnpm ≥ 10.
- Package manager gotcha: lefthook requires `onlyBuiltDependencies: [lefthook]` in `pnpm-workspace.yaml`.
- Exact dependency pinning: where a version is unknown at plan time, install with the tool's `latest`, then PIN the resolved version exactly (`=x.y.z` in Cargo.toml for specta crates; exact semver in package.json for Biome).
- Verification command for every task: `pnpm check:all` must pass before each commit once it exists (Task 5); before that, the task's own verify steps.

/// Migrate any older stored shape to the current schema. Pure fn = unit-testable.
pub fn migrate(value: serde_json::Value) -> Settings {
    let version = value.get("schemaVersion").and_then(|v| v.as_u64()).unwrap_or(0);
    match version {
        CURRENT_SCHEMA_VERSION_U64 => serde_json::from_value(value).unwrap_or_default(),

        _ => {
            let mut s = Settings::default();
            if let Some(t) = value.get("theme").and_then(|v| serde_json::from_value(v.clone()).ok()) {
                s.theme = t;
            }
            s
        }
    }

fn load(app: &tauri::AppHandle) -> AppResult<Settings> {
    let store = app.store(SETTINGS_FILE).map_err(|e| AppError::Store(e.to_string()))?;
    let settings = match store.get("settings") {
        Some(v) => migrate(v),
        None => Settings::default(),

```ts
import { LazyStore } from '@tauri-apps/plugin-store'
import type { StateStorage } from 'zustand/middleware'

const SAVE_DEBOUNCE_MS = 200

export function tauriStoreStorage(fileName: string): StateStorage {
  const store = new LazyStore(fileName)
  let saveTimer: ReturnType<typeof setTimeout> | undefined

  const scheduleSave = () => {
    clearTimeout(saveTimer)
    saveTimer = setTimeout(() => void store.save(), SAVE_DEBOUNCE_MS)
  }

  return {
    getItem: async (name) => (await store.get<string>(name)) ?? null,
    setItem: async (name, value) => {
      await store.set(name, value)
      scheduleSave()
    },
    removeItem: async (name) => {
      await store.delete(name)
      scheduleSave()
    },
  }
}

import { useCallback, useState } from 'react'
import { check, type Update } from '@tauri-apps/plugin-updater'
import { relaunch } from '@tauri-apps/plugin-process'

export function useUpdater() {
  const [status, setStatus] = useState<'idle' | 'checking' | 'available' | 'downloading' | 'ready' | 'upToDate' | 'error'>('idle')
  const [progress, setProgress] = useState(0)
  const [update, setUpdate] = useState<Update | null>(null)

  const checkForUpdates = useCallback(async () => {
    setStatus('checking')
    try {
      const u = await check()
      if (u) { setUpdate(u); setStatus('available') } else setStatus('upToDate')
    } catch { setStatus('error') }
  }, [])

  const install = useCallback(async () => {
    if (!update) return
    setStatus('downloading')
    let total = 0, got = 0
    await update.downloadAndInstall((e) => {
      if (e.event === 'Started') total = e.data.contentLength ?? 0
      if (e.event === 'Progress') { got += e.data.chunkLength; setProgress(total ? got / total : 0) }
      if (e.event === 'Finished') setStatus('ready')
    })
    await relaunch()
  }, [update])

  return { status, progress, version: update?.version, check: checkForUpdates, install }
}
```
