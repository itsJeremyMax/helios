# Security Policy

## Supported versions

Only the latest published release is supported with security fixes. Since
updates ship automatically through the in-app updater, please update before
reporting an issue if you're not already on the latest version.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting a vulnerability

Please do **not** open a public issue for security vulnerabilities. Instead,
use GitHub's private vulnerability reporting:

1. Go to the [Security tab](https://github.com/itsJeremyMax/helios/security)
   of this repository.
2. Click **Report a vulnerability**.
3. Describe the issue, including reproduction steps and impact.

This opens a private advisory visible only to the maintainer and you, so the
issue can be discussed and fixed before any public disclosure.

## Disclosure timeline

We aim to acknowledge new reports within a few business days and to agree on
a fix and disclosure timeline with the reporter. Absent an agreed extension,
we follow a 90-day disclosure window from the initial report: a fix is
released and the advisory is made public within 90 days, or sooner once a
patched release is available.
